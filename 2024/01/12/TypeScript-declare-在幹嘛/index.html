TypeScript 的 declare 在幹嘛

Why ?

如果我們想要在舊的專案引入 typeScript，
應該都會遇到 typescript 對此專案原本那些調用沒有加入 TS 型別註釋的 function 或者 物件的位置回報型別錯誤，
又或者對沒有使用 TS 的外部套件回報型別錯誤。
這個時候我們就需要使用 declare 來對以上這既有的 JS 程式碼做型別定義，好讓 TS 不再報錯。

How

從上面的 Why 我們可以看出來，
會出現沒有做型別定義的地方有兩個:
舊有專案裡面自己定義的方法或物件 和 從 node modules 裡引入的外部 library。

所以，我們需要製作 .d.ts 檔案，並在這些 .d.ts 檔案裡面使用 declare 專門為以上的檔案製作屬於它們的型別定義。

So What ?

可以看出來透過 declare 的功能可以將 型別定義的內容 與 既有的 JS 程式碼 寫在不一樣的檔案中。
並透過 tsconfig.json 檔案的設定來吃到我們用 declare 對於既有程式碼所制定的型別定義，以檢驗我們在使用該功能時是否有傳入與定義的型別相符合的內容。
用這種方式就可以漸漸地讓 TS 慢慢地覆蓋舊有專案，最終讓整個專案都有 TS 的型別檢驗的保護。
所以，在某些專案的 package.json 裡，可以看到以 @type/ 開頭的檔案(e.g. @type/lodash)，這種檔案就是這些 library 自己的 Typescript 型別定義內容。
